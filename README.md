# Persistent Local Kubernetes Infrastructure (WSL2 + KinD)

A collection of architecture patterns, configuration scripts, and systems engineering workflows designed to transform ephemeral local Kubernetes environments into resilient, long-running persistent sandbox environments. 

Developed and maintained by Kevin Rutenberg (Continuous R&D since 2021).

## 🚀 The Core Engineering Breakthroughs

### 1. Zero-Downtime, Persistent Node IPAM for KinD
* **The Problem:** Hard-killing or shutting down WSL2 forces Docker to scramble internal container bridge IP assignments. This breaks the Kubernetes API server's hardcoded TLS certificate loops and destroys cluster access.
* **The Solution:** Implemented a graceful system-level init exit orchestration pattern. By utilizing a controlled logout routine, the Linux kernel cleanly unmounts distributed storage blocks and flushes daemon states smoothly. Docker cleanly caches its IPAM state tables, guaranteeing immutable node IP endpoints upon environment recovery.

### 2. Eliminating VHDX Disk Corruption under K8s I/O
* **The Problem:** WSL2's native `Sparse Mode` allocation struggles with high-frequency write operations generated by active Kubernetes log streams and internal image registries, leading to block-allocation mismatches and file system corruption.
* **The Solution:** Disabled Sparse Mode storage optimization entirely. Replaced with custom, scheduled virtual host disk optimization workflows utilizing upstream Windows PowerShell and `diskpart` virtualization utilities to cleanly compress the underlying `ext4.vhdx` footprint safely.

## 🛠️ Complete Systems Architecture Journey (2021 - Present)

### 🔗 High-Availability Cluster Creation (Kubernetes The Hard Way)
* **Environment:** Google Cloud Platform (GCP)
* **Engineering Focus:** Built a highly available control plane by manually provisioning components from the ground up. Generated custom X.509 TLS certificate authority chains, deployed isolated etcd distributed consensus quorums, and configured manual load balancers for api-server ingress routing.

### 💻 Infrastructure-as-Code Local Multi-Node Deployment
* **Environment:** Vagrant & Oracle VirtualBox
* **Engineering Focus:** Wrote configuration scripts to fully automate local bare-metal node virtual machines. Managed private host-only virtual networking, custom software routers, and Linux kernel package dependencies to establish bare-metal control plane environments before containerization layers were standardized.

### 🐳 Modern DevOps Stack Integrations
The persistent WSL2 core environment currently orchestrates:

* **Automation & CI/CD Engine:** Jenkins LTS (v2.541.x+) instances running Java 21 container runners, utilizing secure **ngrok reverse-proxy tunnels** to expose local webhooks to internet ingress, enabling automated pipeline execution triggered by external GitHub SCM events.
* **Container Core & Runtimes:** High-performance local environments powered natively by Docker Engine (v27.x) and containerd (v1.7.x) system layer abstractions.
* **Orchestration & UI Observability:** A multi-node local KinD cluster (Kubernetes v1.34+) integrated with Portainer and the modernized microservice-managed Kubernetes Dashboard v7.x running parallel Kong API Gateways.
